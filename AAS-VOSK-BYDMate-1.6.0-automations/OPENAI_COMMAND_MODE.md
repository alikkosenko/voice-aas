# Настройка OpenAI-режима

1. Соберите и установите AAS 1.4.0 поверх или вместо 1.3.4.
2. Откройте настройки AAS.
3. В блоке «Понимание команд через OpenAI» вставьте свой API key.
4. Оставьте модель `gpt-5.6-luna` либо укажите доступную вашему проекту модель с поддержкой Structured Outputs.
5. Нажмите «Сохранить».
6. Нажмите «Проверить API».
7. Включите «Использовать AI для понимания команд».

При выключенном AI-режиме AAS работает как версия 1.3.4. При сетевой ошибке или ошибке API приложение пробует обычный локальный парсер.

## Архитектура

```text
кнопка руля
  → Vosk офлайн
  → итоговый текст
  → OpenAI Responses API + строгий JSON Schema
  → список разрешённых VoiceCommand
  → CommandDispatcher
  → Android/BYD действия по очереди
```

Модель не получает доступ к ADB и не может вернуть произвольную shell-команду. Все действия повторно ограничиваются кодом приложения и диапазонами параметров.

## Дополнительные действия 1.4.1

```text
front_defrost_on / front_defrost_off
climate_flow_only_on / climate_flow_only_off
mirror_heat_on / mirror_heat_off
window_vent / window_half
play_youtube
search_music / play_music
```

Диапазон уровней сидений берётся из настройки приложения: 3 или 5.
