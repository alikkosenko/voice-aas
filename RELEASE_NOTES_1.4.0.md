# AAS 1.4.0 — OpenAI command planner

Основа: полностью рабочая версия 1.3.4. Все исправления HTTP-авторизации, ADB/runtime, YouTube и сидений сохранены.

## Новое

- опциональный AI-режим понимания команд через OpenAI Responses API;
- поле для собственного OpenAI API key и модели в настройках;
- API key шифруется Android Keystore и не хранится в исходниках;
- свободные и косвенные фразы: «мне жарко», «душно», «сделай потише»;
- до 12 команд из одной фразы, выполняемых строго по порядку;
- строгий JSON Schema: модель может выбирать только заранее разрешённые действия;
- никакого произвольного ADB, shell, FID или package name из ответа модели;
- автоматический возврат к обычному парсеру 1.3.4 при ошибке API или отсутствии понятного AI-плана;
- кнопка проверки OpenAI API;
- модель по умолчанию: `gpt-5.6-luna`, поле модели можно изменить.

## Не изменено

- Vosk распознаёт звук локально;
- в OpenAI отправляется только итоговый текст распознавания;
- голосовые ответы полностью отключены;
- команды автомобилю по-прежнему проходят через существующий `CommandDispatcher` и проверки безопасности;
- запись в автомобиль по-прежнему требует отдельного разрешения в AAS.

## Примеры

- «Мне жарко» → климат включить → 20 °C → вентилятор 4.
- «Мне холодно» → климат включить → 24 °C → вентилятор 2.
- «Выключи климат, закрой все окна и включи музыку» → три последовательные команды.
- «Найди на YouTube обзор BYD Seal и поставь громкость 40 процентов» → поиск → громкость.

## Ограничение безопасности API key

Шифрование Android Keystore защищает ключ в сохранённых настройках, но не превращает мобильное приложение в безопасный сервер. На рутованной или скомпрометированной мультимедиа клиентский ключ теоретически можно извлечь во время работы. Для публичного распространения рекомендуется серверный прокси.
